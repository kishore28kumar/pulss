## Dark Mode Variants

All themes include optimized dark mode variants with:
- Reduced brightness for eye comfort
- Maintained color relationships
- Enhanced contrast for readability
- Automatic system preference detection

**Dark Mode Colors:**
- Background: Near-black with slight tint
- Foreground: Off-white for reduced eye strain
- Colors: Adjusted for dark backgrounds

   - Navigate to Branding tab
   - Click on your chosen theme

4. **Test:** Preview on both light and dark modes
5. **Save:** Export configuration for backup

## Creating Custom Themes

### Step 1: Define Theme Object

```typescript
const myCustomTheme: ThemeConfig = {
  id: 'my-custom-theme',
  name: 'My Custom Theme',
  description: 'Description of my theme',
  colors: {
    primary: 'oklch(0.55 0.15 280)',
    secondary: 'oklch(0.97 0.01 280)',
    accent: 'oklch(0.65 0.15 120)',
    background: 'oklch(1 0 0)',
    foreground: 'oklch(0.15 0 0)',
    card: 'oklch(1 0 0)',
    border: 'oklch(0.92 0.01 280)',
    muted: 'oklch(0.97 0.01 280)'
  },
  businessTypes: ['general']
}
```

### Step 2: Add to THEME_PRESETS

```typescript
export const THEME_PRESETS: ThemeConfig[] = [
  // ... existing themes
  myCustomTheme
]
```

### Step 3: Test and Deploy

1. Rebuild the application
2. Test in both light and dark modes
3. Verify accessibility (contrast ratios)
4. Deploy to production
